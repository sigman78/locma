E28 · reactive-limits program · pointer action head · promoted 2026-07-19

The dense action head was the bottleneck.

The reactive net read all 155 action logits off one 64-number tower latent (action_net, Linear 64→155) — so per-card information had to survive flatten(1280) → fuse(256) → tower(64) to reach "its" logit. E27 measured the damage: can_item decodes at 0.98 from the raw observation, 0.61 from the towers. E28 replaces that layer with a pointer head — each logit computed from the slot tokens of the cards that action involves — trained at the exact b0k recipe (locma train-zoo --pointer-head, PointerMaskablePolicy). First architecture win after seven training-side nulls; recipe of record on both reactive rungs.

Prior reactive RoR — dense head 0.791 ppo:b0k_sX,ldraft_sX — the b0k trio at the learned draft (E18b/E11). The lens-guarded b0k stack sat at ~0.849.
New reactive RoR — no guards 0.865 ppo:depot:e28p/e28p_sX.zip,ldraft_sX — the pure pointer net already beats the prior guarded record. Gate-2 headroom +0.073, confirm +0.069.
New guarded RoR — lens over pointer 0.908 lppo:e28p s0|s1|s2,ldraft_sX — new reactive summit, above the 0.890 single-critic planner ceiling, 0.018 below the ensemble planner RoR at ~3× lower cost.

Slot layout is fixed and semantic: tokens 0–7 hand, 8–13 my board, 14–19 op board; slot 20 is a virtual all-zeros row for "no source / no target" (Pass, face attacks, untargeted items). Indices: Pass = 0 · Summon = 1+s · Use = 9 + s·13 + tc (tc 0–5 my board, 6–11 op board, 12 none/face) · Attack = 113 + a·7 + tc (tc 0–5 op board, 6 face). Greyed strip cells are illegal in this mock state — the action mask zeroes them either way; the head computes every logit the same way.

The network, before and after

Layer stack shared by b0k and e28p (gray), diverging only at the action head: the dense Linear 64→155 (slate, before) reads latent_pi alone; the pointer head (petrol, after) additionally taps the transformer's per-slot output z through a forward hook — per-card information reaches its logit without passing the flatten→fuse→tower funnel. The value path is untouched.

The action head — dense vs pointer

The only architectural difference between b0k and e28p.

layerout shapeparams

pointer: mlp(155,1)→(155) Linear(196→128), ReLU, Linear(128→1) — ONE shared MLP batched over all actions; final layer orthogonal gain 0.01, matching SB3's near-uniform init25,345

Head params 10k → 25k, but width is not the mechanism: gate 1's capacity-matched control (fresh dense MLP 64→113→155, ~25.0k params) gains nothing over the plain dense head. The gain is structural slot access.

Gate 1 — the BC-agreement cap breaks

Held-out action agreement with the mcts:100 teacher, seeds 0 / 1 per arm. All arms init from b0k_s0, extractor frozen, pi tower + head fine-tuned by masked-CE behavior cloning. The ~0.37–0.39 plateau held through every prior distill attempt.

Item recall was a wash vs the matched control — information access was never the item problem; the gain is broad targeting quality.

Where the rungs stand — avg-hard3 at the learned draft

The guarded pointer stack is the new reactive summit: above the single-critic vbeam planner ceiling, within 0.02 of the 3-critic ensemble planner recipe, at ~0.26 s/game (~3× cheaper than the planner, ~25× cheaper than rbeam).

What did NOT move: items. E27 probes, gate-1 BC and gate-2 PPO all agree — item underuse is consequence valuation (E14a turn-level branching), not access. E30 territory.

Mechanism. Gate 1 (BC, frozen extractor, capacity-matched control): the pointer head breaks the imitation plateau at matched trainable params — 0.423/0.436 agreement vs 0.390–0.392 for a params-matched dense MLP head, both seeds. The gain is NOT concentrated in item or lethal states: slot-addressed logits improve targeting broadly (which attacker, which summon). Gate 2 converted it to play strength at the identical training recipe, so the dense head was a genuine bottleneck, not a BC artifact.

Plumbing. PointerMaskablePolicy builds the standard MaskableMultiInputActorCriticPolicy, then swaps action_net for PointerActionNet. A forward hook on features_extractor.transformer caches the (B,20,64) per-slot output; the head reads it through a closure, so forward(latent_pi) keeps the exact action_net signature and SB3's distribution plumbing is untouched. The optimizer is rebuilt over the final parameter set (the one built by _build() still references the discarded dense head). Save/load round-trips through normal SB3 machinery — e28p checkpoints load with the ordinary ppo: registry spec.

Items, three instruments. E27: can_item representation decays down the trunk but the head gain does not come from item states. Gate 1: item recall a wash vs control (0.289/0.277 vs 0.295/0.282). Gate 2: item rate per opportunity 0.144/0.214 vs b0k's 0.171 — a seed split, not a mechanism. Item plays pay off across the turn's branching; a single forward pass never simulates that. The autoregressive turn-plan head (E30) is aimed at exactly this residual; E29 (trunk conditioning) is the other open lever toward the pure-net 0.890 milestone.
