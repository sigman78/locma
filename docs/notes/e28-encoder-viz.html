The encoder mixes gently — and the card-id embedding learned nothing.

What did TokenSetExtractor's transformer actually learn? Both recipe-of-record extractors (dense-era b0k_s0, pointer-era e28p_s0) opened up on the same live game — 38 decision states of e28p vs scripted — plus their weight matrices. Attention was captured by replicating the encoder forward pass manually (max deviation from the module output: exactly 0). The short answer: attention is near-uniform context pooling over the real cards, the learned slot embeddings carry real zone geometry, and the 16-d card-id embedding is statistically indistinguishable from its random initialization — supporting the netprobe verdict that the trunk re-embeds rather than computes.

Null — at the random-init control Card-id embedding: does it encode card properties? Linear probes sit at or below the majority baseline (type 0.71 vs majority 0.73; cost/attack/defense R² ≤ 0) and match a random-init control. The two independently trained nets' embeddings correlate 0.997 — it never left initialization. Card identity rides in the 17 numeric token features, not the id embedding.
Real — small but structured Slot embeddings: is zone identity learned? Within-zone cosine clearly above cross-zone (b0k: hand·hand +0.26, op·op +0.20, cross ≈ 0). Magnitude is a nudge — row norms ≈ 0.29 vs token norm ≈ 8 after LayerNorm — enough to tag slots, exactly what the flatten-offset design needs. e28p's geometry is visibly weaker.
Diffuse — soft pooling, not pointers Attention: do heads specialize? Normalized attention entropy 0.87–0.88 (1.0 = uniform); mean top weight 0.31 vs 0.14 uniform. Zone preference is token-count base rate (mass÷share 0.8–1.1, all heads, both nets) — no guard-head, no threat-head. The transformer is a gentle context mixer, consistent with "re-embeds, doesn't compute".

Data. One full game, e28p_s0 (deterministic argmax) vs scripted, balanced draft, seed 424242, token-V0 observation — 38 agent decisions, e28p won. The same 38 observation dicts were pushed through BOTH extractors, so every panel is an apples-to-apples comparison on identical inputs. Attention states: 6 decisions spread evenly through the game.

Attention capture. nn.TransformerEncoder does not expose per-head weights, so the encoder forward was replicated manually (self_attn with average_attn_weights=False, then the post-norm residual + FF blocks, eval mode) and asserted against the module output — max |difference| 0.0 on all 38 states, both nets.

Probes. 5-fold CV, ridge regression (R²) and one-vs-rest ridge classification (accuracy) on standardized targets; the random-init control is N(0,1) of the same shape run through the identical pipeline. The 0.997 b0k–e28p embedding correlation follows from both nets sharing RL seed 0 (identical init) and the embedding receiving ~no training signal — consistent with the probes: whatever the nets know about cards, they read from the numeric token features.

Caveats. One game, one seed per net — read the attention panels as representative anecdotes backed by the aggregate stats (entropy, zone-ratio), not as a census. The e28p driver was deterministic, so states skew toward positions a strong net reaches. Cloud/embedding PCAs are per-net (axes are not comparable across nets).
