Inside depot:b0k: where 30,000 game states live in the net

Every panel below is computed from real forward passes of the trained token net (and a same-architecture random re-init) over 30,000 held-out battle decisions. Pick a layer; the spectrum shows how much of its width carries variance (“information density”), the state cloud shows what its top principal components encode, and the unit board shows which individual units are alive, saturated, or idle.

Data. depot:b0k/b0k_s0.zip forwarded over the first 30,000 states of runs/probe-mcts-token.npz (held-out, recorded at seed 1,000,000; mcts:100 teacher vs the 5-opponent roster). PCA sample: 2,600 states, fixed RNG. Critic value = predict_values on the same states.

Layers. slots = transformer per-slot output (flattened, 20×64); features = extractor fused head (input to both towers); pi/vf a1–a2 = post-tanh tower activations; logits = 155-d action scores; raw = the observation itself (tokens + scalars + mask, 373-d).

Reading it. Spectra are eigenvalues of the activation covariance, normalized to sum 1; participation ratio PR = 1/Σp². Unit health uses the netdiag definitions: tanh saturation |a| > 0.99, relu duty = fraction of inputs where the unit is active. CKA is linear (debiased-free), computed on a 4,000-state subsample. Re-init clone uses PyTorch default init — read init curves coarsely.

Provenance. Branch feat/net-probe (merged, PR #83) + this bundle from scripts/netviz_extract.py. Companion page: the held-out verdict (probe matrix over 3 seeds × 2 teachers).
