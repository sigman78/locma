# Policy experiment plans — 2026-07-09

## Current state

The project is at a clear inflection point: stop tuning reactive PPO and move
to opponent-aware, multi-turn planning.

- Current reactive recipe: `ppo:depot:b0k/b0k_sX.zip,depot:ldraft/ldraft_sX.zip`,
  avg-hard3 **0.791**.
- Current practical planner: 3-critic
  `vbeam:depot:shared/shared_s0.zip|depot:shared/shared_s1.zip|depot:shared/shared_s2.zip,8,20,depot:ldraft/ldraft_sX.zip`,
  avg-hard3 **0.978**.
- Learned draft is the latest major RL success: approximately **+0.108** on
  the reactive rung and **+0.052** on the planner rung.
- `edraft` recovers most of the learned-draft gain without inference, but its
  evidence is still pilot-scale.
- Reactive-policy training routes are convincingly exhausted. Larger or
  different PPO, observation changes, opponent diversity, self-play,
  imitation, value fitting, ranking losses, and planner distillation all
  failed or saturated.
- E14/E15 localize the remaining problem to branching and planning, not
  memory or missing observations.
- E22 is the decisive latest result: fair dMCTS beats the 0.978 planner after
  only about 1–2 turns of genuine tree depth. Hidden-information cheating
  contributes little; opponent-aware depth is the lever.
- Search cost remains the main practical constraint. The 3-critic planner is
  about 0.7 seconds/game, while `netdmcts` is about 33 seconds/game at `8x40`
  and 89 seconds/game at `8x160`.

The older result calling `netdmcts:8,40` the strongest policy used the older
`selfplay-r2` oracle and balanced draft. It has not been properly compared
with the current `b0k`/`shared` models and `ldraft`. Its old 0.817 score and
the current planner's 0.978 score come from different configurations and
should not be compared directly.

## Priority 1: E23 — depth allocation in current `netdmcts`

This is the highest-information next experiment and requires little new
policy machinery.

`netdmcts` currently creates `K` independent trees with `I` iterations per
tree. Thus `8x40` means eight very shallow 40-iteration trees. E22 says that
hidden-world coverage is secondary while depth is primary, so the current
allocation may be backwards.

Run a fixed-total-budget sweep:

| K worlds | I per tree | Total evaluations |
|---:|---:|---:|
| 1 | 320 | 320 |
| 2 | 160 | 320 |
| 4 | 80 | 320 |
| 8 | 40 | 320 |
| 16 | 20 | 320 |

### Design

1. Add an optional draft override to the `netdmcts` registry spec. It is
   currently hardwired to balanced draft.
2. Give both sides the same `ldraft_s0` so the experiment isolates battle
   search.
3. Test `b0k_s0` and `shared_s0` as candidate oracles.
4. Compare directly against the 3-critic `vbeam` recipe, not only against
   HARD3.
5. Pilot with 50 mirrored games per cell, then take the best allocation to
   200 games and fresh seeds.
6. Repeat a smaller allocation sweep with heuristic dMCTS to test whether
   the effect is specific to neural PUCT.

### Prediction

`K=1–4` with deeper trees should beat `8x40` at approximately equal neural
inference cost. A null result would instead show that determinization
diversity matters more for net-guided search than E22 suggests.

### Prepared run package

Implementation is ready in `scripts/e23_netdmcts_alloc.py`:

- `netdmcts:K,I,c_puct,MODEL,DRAFT` now accepts the same optional draft
  override as `ppo`/`vbeam`, while historical four-parameter specs retain the
  balanced default.
- Pilot defaults: both `b0k_s0` and `shared_s0`, all five fixed-budget
  allocations, 25 seed pairs / 50 actual mirrored games per cell, common
  random seeds from 27M.
- Confirm defaults: automatically select the pilot winner, then run 100 fresh
  seed pairs / 200 actual games from 28M.
- Every seed block is checkpointed beneath `runs/e23-blocks/`; interrupted
  runs resume without replaying completed blocks.
- JSON summary, Wilson intervals, two-sided binomial p-values, seat splits,
  turn counts, and measured worker-seconds/game are written to
  `runs/e23-summary.json`.
- Dry-run, artifact/device preflight, and an end-to-end smoke mode are built
  in. The smoke completed successfully on the RTX 4080.

Run from the repository root:

```powershell
# Inspect the exact cells and estimated game count.
uv run --extra ml python scripts/e23_netdmcts_alloc.py --dry-run

# Resolve every depot artifact and verify the ML/CUDA environment.
uv run --extra ml python scripts/e23_netdmcts_alloc.py --preflight-only

# Tiny end-to-end check; safe to rerun.
uv run --extra ml python scripts/e23_netdmcts_alloc.py --smoke --stage all

# Real pilot, then fresh-seed confirmation of its winner.
uv run --extra ml python scripts/e23_netdmcts_alloc.py --stage pilot --workers 1
uv run --extra ml python scripts/e23_netdmcts_alloc.py --stage confirm --workers 1
```

The 10-cell pilot is 500 actual games and is estimated at roughly 4.6 serial
GPU-hours from the existing 33 sec/game benchmark. Start at one worker;
compare two workers only by the driver's measured worker-seconds/game because
single-GPU multi-process scaling is not guaranteed.

## Priority 2: build `rbeam` — reply-aware turn beam

This is the recommended new RL-backed policy.

`vbeam` searches only the current player's remaining turn and never crosses
`Pass`. E22 shows that adding even one genuine opponent reply should be
valuable. `rbeam` would perform turn-level expectiminimax:

1. Generate the top several complete own-turn plans using the existing beam.
2. For each plan, sample plausible hidden worlds.
3. End the turn and generate the opponent's strongest complete reply in each
   world.
4. Evaluate the resulting next-turn state with the PPO/shared critic
   ensemble.
5. Select the root plan with the highest mean value across worlds.

This remains fair: the root action is shared across determinizations, while
the opponent adapts to the hand it actually knows in each sampled world.

### Why this policy is attractive

- It directly targets the missing depth identified by E22.
- It searches at turn granularity, where the meaningful tactical choices
  live.
- Neural evaluation can be batched across candidate plans and worlds.
- It should occupy the cost gap between `vbeam` and `netdmcts`.
- It reuses the strongest confirmed assets—`shared` critics and `ldraft`—with
  no retraining required.

Initial success gate: greater than 55% head-to-head against the current
`vbeam` recipe while remaining below roughly 5–10 seconds/game.

## Priority 3: replace the saturated evaluation ruler

Avg-hard3 is no longer sufficient for top-end promotion. A policy at 0.978
has only +0.022 numerical headroom, making the historical +0.03 promotion
threshold impossible.

Create an advanced ruler containing:

- Current 3-critic `vbeam + ldraft`.
- Fair dMCTS around its crossover point, `15x100 + ldraft`.
- The best E23 `netdmcts` configuration.
- The boardkeep/exploit suite.
- A direct head-to-head matrix plus turns/sec and seconds/game.

Use identical drafts for battle-skill experiments. Separately evaluate each
complete policy with its preferred draft. Direct matches are important
because the policy pool is strongly non-transitive.

## Secondary experiments

### Fully confirm `edraft`

`edraft` matched learned draft under the planner at pilot scale. Run the full
40x25 evaluation and a fresh-anchor confirmation on both reactive and planner
rungs. A pass would justify promotion as the zero-inference deployment
drafter.

### Retrain the `shared` critic family on learned-draft decks

E19 found a real but small +0.010 critic gain from deck-matched training.
Revisit this only after the advanced ruler exists; HARD3 cannot reliably
measure improvements above the current 0.978 planner score.

### AlphaZero/self-play v2

Revisit search training only after E23 or `rbeam` produces a stronger search
teacher. Do not rerun the old Phase-2 loop unchanged: it used fragmented
shallow search, only 100 self-play games per iteration, and a noisy 16-game
gate. A future version should use:

- The winning search geometry from E23 or `rbeam`.
- Current `b0k`/`shared` models and `ldraft`.
- Substantially more search-generated data.
- Multi-turn backed-up values that preserve sibling ordering.
- Paired or sequential evaluation with a properly powered gate.

## Directions not recommended

Do not spend more cycles on ordinary PPO hyperparameter tuning, another
opponent-zoo variation, reactive planner distillation, recurrent memory, or
another observation redesign. The existing evidence closing those directions
is substantially stronger than the evidence supporting another retry.

## Recommended sequence

1. Implement the `netdmcts` draft override and run E23.
2. Establish the advanced evaluation ruler.
3. Build and benchmark `rbeam` on the strength/cost Pareto frontier.
4. Fully confirm `edraft` as low-risk deployment cleanup.
5. Consider AlphaZero/self-play v2 only if the improved search policy creates
   a materially stronger teacher.
